What it is
Conversation first

Claude + Claude Code

Claude is a broad AI assistant for reasoning, writing, research, and creation. Claude Code brings the same product family into repositories to inspect, edit, run, test, and ship software.

Execution first

A coding agent

Codex is OpenAI’s software-development agent. It works directly with repositories and tools, can coordinate subagents, and returns changes and evidence for review.

Best fit

Collaborative exploration

Strong when the work moves between ideas, writing, research, visual feedback, product decisions, and interactive implementation.

Bounded repository work

Strong when a concrete engineering task should be planned, delegated, implemented, tested, reviewed, and prepared for delivery.

Published context

Up to 1,000,000 tokens

Fable 5, Opus 4.8, and Sonnet 5 list 1M-token context windows. Haiku 4.5 lists 200K. Maximum synchronous output is 128K for the three larger models and 64K for Haiku 4.5.

Product usage limits and long-session compaction remain separate from a model’s API context specification.

Up to 1,050,000 tokens

GPT‑5.6 Sol, Terra, and Luna each list a 1.05M-token context window and 128K maximum output. Spark is a separate Pro research preview; its public Codex guide does not publish an equivalent context figure.

Repository retrieval, compaction, task limits, and subscription usage are separate from model context.

Parallel agents

Agent teams and custom agents

Claude Code can spawn multiple agents for parallel work and supports custom agents, shared project guidance, hooks, and skills. Availability and UI vary by surface.

Subagent workflows

Codex can delegate independent work to subagents in the app, CLI, and IDE. Custom agents can carry specialized instructions, models, reasoning levels, and sandbox settings.

Durable context

CLAUDE.md

Stores project instructions and memories. Claude Code settings, hooks, MCP servers, skills, and custom commands extend the working environment.

AGENTS.md

Stores repository layout, commands, conventions, constraints, and verification rules. Nested files can narrow guidance for a subtree.

Permissions & safety

Permission modes and hooks

Claude Code can plan without editing, request approval for actions, and enforce repeatable checks through hooks and project settings.

Approval mode and sandbox mode

Codex separates when it asks for permission from what the operating sandbox can access. Subagents inherit the parent turn’s live policy unless explicitly constrained further.

Extensibility

MCP servers, connectors, hooks, skills, plugins, custom commands, and integrations connect Claude Code to external systems and repeatable team workflows.

MCP servers and app connectors provide live external access; skills package workflows; plugins bundle reusable capabilities; hooks enforce lifecycle checks; automations schedule stable work.
