01The four everyone should know

Stay in the driver's seat.

These four turn Claude Code from "hope it goes well" into "I'm steering." Learn only these and you're ahead of most people.

Claude Code simulation

Plan mode · Shift+Tab

Think first, build second

Press Shift+Tab to cycle into Plan mode (or type /plan). Claude explores and proposes an approach without editing a thing until you approve.

Reach for it when: the task is bigger than a one-liner.
/rewind · Esc Esc

Undo a wrong turn

Every message you send is a save point. Press Esc twice or type /rewind to travel back — restore the code, the conversation, or both.

Reach for it when: "I liked it better a few steps ago."
/compact

Reclaim breathing room

Long sessions fill up Claude's memory of the chat. /compact summarizes what's happened so far so you can keep going without starting over.

Reach for it when: a session gets long and sluggish.
/fewer-permission-prompts

Stop being asked twice

Tired of approving the same safe action repeatedly? This scans your session and offers to pre-approve the harmless, repetitive ones.

Reach for it when: permission pop-ups start to nag.
02Quality on demand

Let Claude check its own work.

You don't need to be able to read the code to ask for a review. Claude can grade its own changes — and even send a whole crew of reviewers into the cloud.

/code-review

A second pair of eyes

Reviews the changes in your current session for bugs and rough edges, and offers to fix what it finds. Fast, and free with your plan.

Reach for it when: before you call something "done."
/code-review ultra

Send in the whole crew

Launches a fleet of independent reviewers in the cloud to hunt harder for bugs. It's a paid, deeper pass (billed per run) for when it really matters.

Reach for it when: shipping something people depend on.
/security-review

Check the locks

Looks specifically for security problems in your pending changes — leaked secrets, unsafe input, risky permissions.

Reach for it when: anything handles real people's data.
/verify

Actually try it

Goes beyond "the tests pass" and drives the real feature end-to-end to confirm it behaves the way you meant.

Reach for it when: you want proof, not a promise.

Make /code-review a reflex at the end of every change — it's the cheapest quality you'll ever buy.

03Do more without doing it yourself

Set goals, repeat work, run on a schedule.

These hand Claude a bigger slice of the job — a target to keep working toward, a task to repeat, or a whole app to launch and test.

/goal

Keep going until it's done

Give Claude a finish line — "make every page work on mobile" — and it keeps working toward that goal instead of stopping after one step.

Reach for it when: the job needs several rounds.
/run

Launch & drive your app

Starts your project and clicks through it so you can see the change working for real — not just a promise that it does.

Reach for it when: "show me it actually runs."
/loop

Do it on repeat

Runs a prompt or command again on an interval — "re-run the tests every 10 minutes" — or lets Claude pace itself.

Reach for it when: you're watching for something to change.
/schedule

Wake it up later

Creates a recurring cloud task — a "routine" — that runs on a schedule even when your laptop is closed.

Reach for it when: "every morning, do this for me."
/deep-research

Send a research party

Fans out across many sources, checks its own claims, and comes back with a cited report — great before a big decision.

Reach for it when: you need answers you can trust.
/simplify

Tidy up afterwards

Reviews recent code purely for cleanups — shorter, clearer, less repetition — and applies the improvements.

Reach for it when: it works, but it got messy.
⚡The pattern behind all of these: describe the outcome you want, hand Claude room to work, then check the result. Bigger commands don't mean less control — you still approve what matters.
04Housekeeping worth knowing

Set things up and keep them tidy.

/init

Teach the project

Creates a CLAUDE.md — a note Claude reads at the start of every session — describing your project's rules and how to run it.

New project? Run this first.
/memory

Edit what it remembers

Opens the notes Claude keeps about your work so you can add, correct, or remove them.

When it keeps forgetting or misremembering.
/model

Swap the brain

Switch models mid-session — a faster one for simple edits, the most capable for hard problems.

Hard problem? Reach for the strongest model.
/mcp

Manage connections

See and reconnect the outside tools Claude is linked to — GitHub, Slack, your calendar, and more.

When a connected tool stops responding.
/chrome

Give Claude a browser

Connects the Chrome integration so Claude can click through your live app like a real tester.

When you want it to test what you see.
/clear

Fresh start

Begins a clean conversation. The old one isn't lost — you can still find it in your sessions.

When you switch to a totally new task.
/config

Open settings

Jump to Claude Code's settings to adjust preferences without hunting through menus.

When you want to change how it behaves.
/help

The master list

Shows every command available to you right now. When in doubt, this is home base.

Always safe to run. Start here.
05No slash required

Four moves that aren't commands at all.

Some of the most useful power moves are just things you can do in the message box — and beginners almost never realize they exist.

＠
@ a fileType @ and a filename to pull that file into the conversation — no copy-paste.
🖼️
Paste a screenshotDrag or paste an image of what looks wrong. Claude can see it and fix from the picture.
👁️
Ctrl+O viewsCycle Normal, Verbose, and Summary to see as much — or as little — of Claude's work as you want.
＋
The + buttonBeside the message box: attach files, add Skills, connect tools, and install plugins.
